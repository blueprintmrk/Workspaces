<h2 class="answer-title">
Signing in using application-specific passwords
</h2>
<div class="clear"></div>
</div>
<div class="article_content article-content-185833" id="article-content-div">
<p>The codes that you’ll get with 2-step verification don’t work with all applications. There are a few applications that use information from your Google Account (like the Gmail application on your phone, or Outlook) that require another type of code. These are called <em>application-specific passwords</em>. </p>
<p>Some devices and applications that use these special passwords include:</p>
<table tableborder="0" padding="5">
<tbody><tr>
<td>
<ul>
<li>POP and IMAP email clients such as Outlook, Mail and Thunderbird </li>
<li>Gmail and Google Calendar on smartphones </li>
<li>ActiveSync for Windows Mobile and iPhone </li>
<li><a href="//www.google.com/support/youtube/bin/answer.py?answer=184968">YouTube Mobile on Apple devices</a> </li>
<li><a href="//www.google.com/support/cloudprint/?expand=twostep#twostep">Cloud Print</a> </li>
</ul>
</td>
<td>
<ul>
<li>Installed chat clients such as <a href="//www.google.com/support/talk/bin/answer.py?answer=23944">Google Talk</a> and Adium </li>
<li>3D Warehouse, Sketchup, and installed applications </li>
<li>AdWords Editor </li>
<li><a href="//www.google.com/support/chrome/bin/answer.py?answer=1181420">Sync for Google Chrome</a> </li>
<li><a href="https://mail.google.com/support/bin/answer.py?hl=en&answer=9429">Gmail Notifier</a></li>
</ul>
</td>
</tr>
</tbody>
</table>
<h4>How to generate an application-specific password</h4>
<p>Unlike a verification code that you receive from your phone or through Google Authenticator, application-specific passwords are <em>generated</em>. You don’t have to memorize them, ever - you simply generate a new one each time you need to sign into a particular application. Follow the steps below to generate your application-specific password: </p>
<ol>
<li>Visit the <a href="https://www.google.com/accounts/IssuedAuthSubTokens" target="_blank">Authorizing applications & sites page</a> <img src="//services.google.com/images/adwords/doit.gif" alt="new window"> (pictured below) under your Google Account settings. </li>
<li>Under the <strong>Application-specific passwords</strong> section, enter a descriptive name for the application (label 1) you want to authorize, such as "AdWords Editor – Desktop," then click “Generate application-specific password" (label 2).</li>
</ol>
<p><img src="//www.google.com/help/hc/images/accounts/accounts-2step-ASP-create.png" alt="application-specific password creation" /></p>
<p>You'll then see the application-specific password you just created (3). You'll also see the name you wrote in for the device (4) and a link to <strong>Revoke</strong> (5) -- or cancel -- the code.</p>
<img src="//www.google.com/help/hc/images/accounts-2step-ASP-generated.gif" alt="application-specific password example" />
<p>Once you click <strong>Done</strong>, you’ll never see that application-specific code again. But don't panic: You can generate a new one whenever you’re prompted for an application-specific password by a non-browser device or application -- even a device or application you’ve authorized before.</p>
<h4>Using application-specific passwords</h4>
<p>When prompted for a password when you sign in to a non-browser application or device that accesses your Google Account:</p>
<ol>
<li>Enter your username.</li>
<li>Enter your application-specific password in the password field.</li>
<li>If your application has an option to remember your application-specific password or stay signed in, you can select that option so you won’t have to generate and enter a new application-specific password each time you access your account from this application or device.</li>
</ol>
<div class="note"><a href="//www.google.com/support/chrome/bin/answer.py?answer=1181420">For sync on Google Chrome</a>, don’t enter the application-specific password in the password field. You’ll be prompted to enter the application-specific password on the next screen.
</div>
<h4>Examples of using application-specific passwords</h4>
<p></p><a class="zippy">Apple Mail</a>
<div>
<p>Soon after you turn on 2-step verification, you'll be alerted that your password is no longer working (see image below). Enter an application-specific password that you generated in the "Password" field (1). If you want Mail to store your password for future sign in, you can click the box next to "Remember this password on my keychain" (2). </p>
<p><img src="//www.google.com/help/hc/images/accounts-2step-mail-challenge.jpg" alt="application-specific password Mail example" /> </p>
</div>
<p></p><a class="zippy">Android on Nexus S</a>
<div>
<p>Soon after you turn on 2-step verification, you'll be alerted that your password is no longer working (see image below). Enter an application-specific password that you generated in the "Password" field (1).</p>
<p><img src="//www.google.com/help/hc/images/accounts-2step-and-mail-challenge.jpg" alt="application-specific password Android example" /> </p>
</div>
<br /> <br />
<div class="note">
Third-party applications that use Google's API feed, such as Blogger plug-ins, might not accept 2-step verification codes. Check the resources provided by the plug-in for the best way to use it with 2-step verification.
</div>
